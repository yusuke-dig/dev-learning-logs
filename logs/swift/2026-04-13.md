### StreakCalculatorのロジック修正

#### ループの役割

```swift
var streak = 0
for daysAgo in daysAgoArray {
    if daysAgo == streak {
        continue       // 同じ日に複数記録 → スキップ
    } else if daysAgo == streak + 1 {
        streak += 1    // 次の日の記録 → streak増加
    } else {
        break          // 連続が途切れた → ループ終了
    }
}
```

`streak` は「何日連続か」ではなく**「何日前まで遡ったか」**を表している。

---

#### なぜ `if daysAgoArray.first == 0` が必要か

今日の記録（`daysAgo=0`）は `continue` でスキップされるため、ループだけでは今日の1日分が加算されない。

```swift
// daysAgoArray = [0, 1, 2]（今日・昨日・一昨日）
// ループ後: streak = 2（本来は3のはず）

if daysAgoArray.first == 0 {
    streak += 1  // 今日の記録がある場合だけ補正
}
// streak = 3 ✅
```

---

#### テストケースで確認

| days | daysAgoArray | ループ後 | 補正 | 結果 |
|---|---|---|---|---|
| `[-2, -1, 0]` | `[0, 1, 2]` | 2 | +1 | 3 ✅ |
| `[-3, -2, -1]` | `[1, 2, 3]` | 3 | なし | 3 ✅ |
| `[-2]` | `[2]` | 0 | なし | 0 ✅ |
| `[0]` | `[0]` | 0 | +1 | 1 ✅ |

---

#### まとめ

```
今日の記録がある（first == 0）
　→ continueでスキップされた今日の分を最後に+1

今日の記録がない（first != 0）
　→ 補正不要
```

`continue` の設計上の副作用を、ループの外で補正するパターン。

### Swiftのテストの書き方

```swift
@Test("Streak calculations", arguments: [
    Input(expectedStreak: 0, days: []),
    Input(expectedStreak: 1, days: [0]),
    ...
])
func testCalculations(input: Input) {
    let streak = streakCalculator.calculateStreak(for: moments)
    #expect(streak == input.expectedStreak, "\(input.days)")
}
```

- `@Test` → テストメソッドのマクロ
- `arguments` → 複数のテストケースをまとめて渡せる（パラメータ化テスト）
- `#expect` → 期待値と実際の値を比較するアサーション
- `@testable import` → `internal` なクラスや関数もテストから参照できるようにする
