## 今日学んだこと

### `private` の意味と必要性

```swift
private var contentStack: some View { ... }
```

- なくても動くが、「このViewの内部実装である」という意図をコードで伝えるために付ける
- Swiftの設計思想：「デフォルトで閉じて、必要なものだけ開く」

```
private  → このファイル内のみ
internal → モジュール内（デフォルト）
public   → 外部からもアクセス可能
```

---

### 計算プロパティ（Computed Property）とメソッドの使い分け

```swift
// 計算プロパティ → 引数なし・固定の内容を返す
private var contentStack: some View {
    VStack { ... }
}

// メソッド → 引数あり・内容が変わる
func header(_ text: String) -> some View {
    Text(text)
}
```

「再計算が必要かどうか」ではなく「**引数が必要かどうか**」で使い分ける。

`body` 自体もコンピューテッドプロパティで、`get { }` が省略されているだけ。

```swift
// この2つは同じ意味
var body: some View {
    Text("Hello")
}

var body: some View {
    get { Text("Hello") }
}
```

### `_` による引数ラベルの省略

```swift
func header(_ text: String) -> some View
//          ^
//          引数ラベルを省略できる

header("Your Badges")    // ラベルなしで呼び出せる
header(text: "Badges")   // _ がなければラベルが必要
```

### `@Query` のフィルター

```swift
@Query(filter: #Predicate<Badge> { $0.timestamp != nil })
private var unlockedBadges: [Badge]
```

- `<Badge>` という型指定でBadgeテーブルを自動参照
- `[Badge]` という変数の型でもどのテーブルか決まる
- `@Model` を付けたクラスが自動でテーブルとして登録されているため

SQLで例えると：
```sql
SELECT * FROM badges WHERE timestamp IS NOT NULL;
```

### タプルによる複数キーソート

```swift
unlockedBadges.sorted {
    ($0.timestamp!, $0.details.title) < ($1.timestamp!, $1.details.title)
}
```

タプル比較は**左から順番に比較する**ルール。

```
① まず timestamp で比較
② timestamp が同じだったら title で比較
```

SQLで例えると：
```sql
ORDER BY timestamp ASC, title ASC;
```

`timestamp!` と強制アンラップできる理由は、`timestamp != nil` でフィルター済みなので必ずnilでないことが保証されているため。

### `Int` 準拠がソートに活きる

enum BadgeDetails: Int {

最初に `Int` を準拠させた設計がソートにも活きている。

### `.scrollClipDisabled()`

ScrollViewの境界でコンテンツがクリップ（切り取り）されるのを無効にするモディファイア。

```swift
ScrollView(.horizontal) {
    HStack { ... }
}
.scrollClipDisabled()       // 影やアニメーションがはみ出しても切れない
.scrollIndicators(.hidden)  // はみ出しによるスクロールバーの崩れを防ぐ
```

セットで使うのが一般的。

### 六角形へのバッジ配置（三角関数）

```swift
private var yOffset: CGFloat {
    let radius = hexagonLayout.size / 2
    let yOffsetFromHexagonCenter = sin(Angle.degrees(30).radians) * radius
    return radius - yOffsetFromHexagonCenter - (size / 2)
}
```

正六角形の幾何学的性質から `sin(30°) = 0.5` を使って中心から辺までの距離を計算し、下の頂点にバッジを正確に配置するためのオフセット計算。

```
六角形の中心
　↓ radius（半径分下へ）
六角形の一番下
　↑ yOffsetFromHexagonCenter（辺の位置まで戻る）
平らな辺の位置
　↓ size / 2（バッジ画像の中心を合わせる）
バッジ画像の中心位置
```
