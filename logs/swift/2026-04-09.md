### `enum` の継承（`Int, Codable, CaseIterable`）

```swift
enum BadgeDetails: Int, Codable, CaseIterable
```

| 指定 | 目的 |
|---|---|
| `Int` | 各caseに整数を自動割り当て（firstEntry=0, fiveStars=1...） |
| `Codable` | JSONへの変換・復元を自動化（SwiftDataやAPIで保存できる） |
| `CaseIterable` | `allCases` で全caseを配列として取得できる |

3つセットで「保存・表示・一覧取得」をまとめて対応する。

### `@Model` class はDBのテーブル定義

```swift
@Model
class Badge {
    var details: BadgeDetails  // カラム定義
}
```

PostgreSQLで例えると…

```sql
CREATE TABLE badges (
    id UUID PRIMARY KEY,  -- @Modelが自動付与
    details INTEGER       -- BadgeDetails は Int なので INTEGER
);
```

`extension` でサンプルデータを分離するのは「モデルの定義」と「サンプルデータ」の**関心の分離**のため。

### `FetchDescriptor` とは

SwiftDataの検索条件オブジェクト。

```swift
var fetchDescriptor = FetchDescriptor<Badge>()
fetchDescriptor.fetchLimit = 1
let existingBadges = try context.fetch(fetchDescriptor)
```

SQLで例えると：
```sql
SELECT * FROM badges LIMIT 1;
```

`fetchLimit = 1` にするのは存在確認だけなら1件で十分なため（効率化）。

### `loadBadgesIfNeeded()` の命名の意図

```swift
func loadBadgesIfNeeded()
//              ^^^^^^^^ 「必要なときだけ」ロードする
```

- `loadBadges()` → 毎回実行する印象
- `loadBadgesIfNeeded()` → 必要なときだけ実行する意図が名前で伝わる

Appleの `layoutIfNeeded()` なども同じ命名パターン。**名前でコードの意図を伝える**ことが大事。

実装の中身：

```swift
if existingBadges.isEmpty {
    // まだ保存されていない → 全バッジをINSERT
    for details in BadgeDetails.allCases {  // CaseIterableが活きる
        context.insert(Badge(details: details))
    }
}
// すでにある → 何もしない（IfNeededの部分）
```

### 責務の分離（単一責任の原則）

```swift
// DataContainer は「呼ぶだけ」
try badgeManager.loadBadgesIfNeeded()
```

```
DataContainer → アプリ全体のセットアップだけ責任を持つ
BadgeManager  → バッジの管理だけ責任を持つ
```

メリット：バッジの初期化ロジックが変わっても `BadgeManager` だけ修正すればよく、`DataContainer` は触らなくていい。

PostgreSQLでいうと「テーブルごとにリポジトリクラスを分ける」のと同じ発想。ソフトウェア設計の**単一責任の原則（SRP）** に対応する。

### `.clipShape` vs `Rectangle` から作る

```swift
// ✅ おすすめ：clipShape
VStack { ... }
    .background(color)
    .clipShape(RoundedRectangle(cornerRadius: 16))

// ❌ 避ける：Rectangleから作る
ZStack {
    RoundedRectangle(cornerRadius: 16).fill(color)
    VStack { ... }
}
```

SwiftUIの設計思想は「**コンテンツを定義してから見た目を装飾する**」なので `clipShape` の方が自然。`Rectangle` から作るのはUIKitに近い古いやり方。

### `.multilineTextAlignment` とは

複数行になったテキストの揃え方を指定するモディファイア。CSSの `text-align` と同じ。

```swift
.multilineTextAlignment(.center)  // 中央揃え
.multilineTextAlignment(.leading) // 左揃え（デフォルト）
```

1行のテキストには効果がなく、**複数行になって初めて効果が出る**。
