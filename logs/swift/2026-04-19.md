## 今日学んだこと

### `var` と `let` の使い分け

Swiftでは値が変わる可能性があるときに `var`、変わらないときに `let` を使う。
コンパイラが `var` を使ったのに一度も変更しなかった場合に警告を出す設計になっており、`let` を積極的に使うことが推奨される。

```swift
var greeting = "Hello"  // 後から変更できる
let pi = 3.14159        // 変更しようとするとコンパイルエラー
```

### 文字列補間（String Interpolation）

Swiftでは `\()` を使って文字列の中に変数や式を埋め込める。

```swift
let name = "Alice"
return "Hello, \(name)!"  // → "Hello, Alice!"
```

`+` での連結よりも可読性が高く、Swiftでは標準的な書き方。

### Bool を返す式の書き方

比較演算子の結果はそのまま `Bool` になるため、`return (number % 2 == 0)` のように括弧で囲む必要はない。

```swift
// 冗長
return (number % 2 == 0)

// Swiftらしい書き方
return number % 2 == 0
```

`Bool` を返す関数では括弧を省くのが慣習。

### `@testable import` とテストの書き方

Swiftのテストでは `@testable import` を使うことで、`internal` なシンボルにアクセスできる。
テストのマクロ構成は以下の通り。

```swift
import Testing
@testable import SwiftLearning

@Suite("Day01: 変数と定数")
struct Day01Tests {
    @Test("greet は Hello, {name}! を返す")
    func testGreet() {
        #expect(greet("Alice") == "Hello, Alice!")
    }
}
```

- `@Suite` → テストのグループを定義するマクロ
- `@Test` → テスト関数を定義するマクロ
- `#expect` → 期待値と実際の値を比較するアサーション
