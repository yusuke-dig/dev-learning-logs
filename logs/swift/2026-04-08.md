### `.enumerated()` とは

配列の要素とインデックス（番号）をセットで取得するメソッド。

```swift
// 通常のForEach（indexが使えない）
ForEach(moments) { moment in ... }

// .enumerated()でindexも使える
ForEach(moments.enumerated(), id: \.0) { index, moment in ... }
```

- `id: \.0` → タプルの最初の要素（index）をIDとして使う指定
- Pythonの `enumerate()` と同じ概念

TypeScriptの `forEach` は最初からindexが使えるが、SwiftUIの `ForEach` はindexより**要素のID**を重視した設計のため、indexが必要なときだけ `.enumerated()` で取り出す。

---

### SwiftUIのForEachがIDを重視する理由

```
// indexで管理する場合
// → 要素を削除するとindexがずれてSwiftUIが誤検知し、アニメーションが崩れる

// IDで管理する場合
// → 何番目になっても同じ要素と認識できる（安定）
```

ReactもIDを使うべきという考え方は同じだが…

| | TypeScript/React | Swift/SwiftUI |
|---|---|---|
| 間違いを防ぐタイミング | 実行時 | コンパイル時 |
| ID管理 | 推奨だが強制はしない | APIレベルで強制 |
| 基本思想 | 柔軟性・表現力 | 安全性・堅牢性 |

### `.scrollTransition` とは

スクロール中にViewが画面に入ってくる・出ていくときのアニメーションを定義するモディファイア。

```swift
.scrollTransition { content, phase in
    content
        .opacity(phase.isIdentity ? 1 : 0)    // 画面外では透明
        .scaleEffect(phase.isIdentity ? 1 : 0.8) // 画面外では小さい
}
```

- `phase.isIdentity` が `true` → Viewが画面内に完全に表示されている状態
- `phase.isIdentity` が `false` → 画面外（入ってくる途中 or 出ていく途中）
- `ScrollView` の中のViewにしか効果がない
- `NavigationLink` に限らずどんなViewにも使える

---

### `.defaultScrollAnchor` とは

ScrollViewのスクロール位置を制御するモディファイア。3つセットで使う。

```swift
.defaultScrollAnchor(.bottom, for: .initialOffset)
// 起動時・画面表示時 → 一番下からスタート

.defaultScrollAnchor(.bottom, for: .sizeChanges)
// コンテンツが追加されてリストが伸びたとき → 自動で一番下にスクロール

.defaultScrollAnchor(.top, for: .alignment)
// コンテンツが少なくスクロール不要なとき → 上から並べる
```

LINEやiMessageのように「起動したら最新が見える・新着が来たら自動スクロール」というUXを実現する。コンテンツ量に応じて自動で切り替わる。

### `.mask` とは

Viewの表示範囲を指定した形でくり抜くモディファイア。

```swift
.mask {
    Image(systemName: "hexagon.fill")  // この形でくり抜く
}
```

- マスクの白い部分 → 表示される
- マスクの透明部分 → 非表示になる

CSSの `clip-path` や `mask-image` に近い概念。

`.clipShape` との違い：

```swift
// clipShape → 決まった形（円、角丸など）でくり抜く
.clipShape(Circle())

// mask → 任意のViewの形でくり抜く（より柔軟）
.mask {
    Image(systemName: "hexagon.fill")  // 画像やViewの形でくり抜ける
}
```

六角形のような特殊な形は `.clipShape` では表現しにくいため `.mask` を使う。

---

### Hexagonの重ね順まとめ

```
① ZStack（背景画像 + content）
　↓ .mask で hexagon.fill の形にくり抜く
② 六角形の形に切り取られる
　↓ .background で背後に敷く
③ hexagon（枠線）が後ろに見える
```
