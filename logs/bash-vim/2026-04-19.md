## 今日学んだこと

### Vimの移動コマンド（hjkl系）

Vimの移動はキーボードから手を離さないことが前提の設計になっている。
矢印キーを使わず、ホームポジションのまま操作することが効率化の出発点。

| コマンド | 動作 |
|---------|------|
| `h` / `l` | 左 / 右に1文字 |
| `j` / `k` | 下 / 上に1行 |
| `w` / `b` | 次 / 前の単語の先頭へ |
| `0` / `$` | 行頭 / 行末へ |
| `gg` / `G` | ファイルの先頭 / 末尾へ |

`w`・`b` を使うと1文字ずつ動く必要がなくなり、単語単位でコードを読み飛ばせる。
`0` と `$` は行内の端への一発移動なので、行が長いほど効く。

### Bashの変数と echo

Bashの変数代入で最も引っかかりやすいのは **`=` の前後にスペースを入れてはいけない**という制約。

```bash
name="Alice"   # 正しい
name = "Alice" # エラー（コマンドとして解釈される）
```

変数を参照するときは `$` をつける。文字列の中に埋め込むときは `${}` で明示するのが安全。

```bash
echo "Hello, ${name}!"  # 変数の境界が明確になる
```

コマンドの実行結果を変数に入れる `$()` は、外部コマンドとの連携の基本形。

```bash
today=$(date +%Y-%m-%d)
```

数値計算は `$(())` を使う。単純な四則演算はこれだけで足りる。

```bash
result=$((num_a + num_b))
```

### VimとBashを組み合わせて学ぶ理由

Bashスクリプトを書くという行為は、実際にはテキストファイルをエディタで編集することそのもの。
「Bashを学ぶ = Vimを使って `.sh` ファイルを編集する」を最初から習慣にすることで、
操作の文脈が統一されて両方が定着しやすくなる。

Vimに `/ + キーワード` で検索できることを知っていると、
演習ファイル内の `# YOUR CODE HERE` を一瞬で見つけられる。
ツールの知識が学習効率に直結する典型例。
