## 今日学んだこと

### Vim: `:s` 置換コマンド

`:s/old/new/` で現在行の最初のマッチを置換、`g` フラグを付けると行内の全マッチを置換する。
`:%s/old/new/g` でファイル全体に適用できる。`c` フラグを追加すると1件ずつ確認しながら置換できる。

変数名を変えたいときに `:%s/old/new/g` を使うと、手動で探して書き直すより速く、見落としもなくなる。
これがVim上でのリファクタリングの基本パターン。

### Bash: `while` 文の2パターン

カウンタで回す場合:
```bash
i=1
while [ $i -le 5 ]; do
  echo "$i"
  i=$((i + 1))
done
```

`read` で標準入力を1行ずつ処理する場合:
```bash
while read line; do
  echo "$line"
done <<EOF
apple
banana
EOF
```

`while read line` はヒアドキュメントやパイプで入力を渡さないと stdin 待ちになって止まる。
`for` と違い入力が尽きたときに終了するので、入力ソースを必ずセットで考える必要がある。

### Bash: `$` の役割

演習中に `$(line)` と書いてしまい、コマンドとして実行されてエラーになった。
この気づきから `$` の役割を整理できた:

- `$var` → 変数の値を取り出す
- `$(cmd)` → コマンドの出力を取り出す（コマンド置換）
- `$((式))` → 算術計算の結果を取り出す（算術展開）

`$` は「値として取り出す」という印で、括弧の種類で何を展開するかが決まる。
`$` なしの `(cmd)` や `((式))` は実行するだけで値は取り出さない。

### Bash: `"$var"` とクォーティング

数値の変数なら `$i` でも `"$i"` でも結果は同じ。
ただしスペースを含む値（ファイル名、ユーザー入力など）では `$var` が単語分割されてバグる。
最初から `"$var"` と書く癖をつけておくのがベストプラクティス。
